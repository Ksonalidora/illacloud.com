## <h2 hidden>Was ist REST API?</h2>

In ILLA Cloud ermöglicht die REST-API-Funktionalität Entwicklern die Interaktion mit externen Systemen Dienstleistungen und Datenquellen unter Verwendung des architektonischen Stils der Repräsentationalen Staatsübertragung (REST) zu verwenden. Mit der REST API in ILLA Cloud können Entwickler ihre Low-Code-Anwendungen nahtlos in andere Anwendungen, Datenbanken oder Dienste integrieren.

Die REST API in ILLA Cloud erlaubt die Erstellung benutzerdefinierter Endpunkte, die HTTP-Anfragen wie GET, POST, PUT und DELETE bearbeiten können. Dies ermöglicht Entwicklern verschiedene Operationen auszuführen, darunter das Abrufen von Daten, das Einreichen von Daten, Aktualisieren von Datensätzen und das Löschen von Informationen.

### REST API erstellen

Es gibt zwei Möglichkeiten, nach der Anmeldung in Ihrem Illa-Konto eine Ressource in Illa zu erstellen.

-   In Ressourcen erstellen

Melde dich in dein Illa Konto an, wähle `**Ressourcen**` oben auf der Seite und klicke auf `**Neu erstellen**` Button.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Wähle **`REST API`** aus der Datenbankliste.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Verbinden Sie sich mit den erforderlichen Parametern, die in `Verbindungseinstellungen` unten beschrieben sind.

Klicke auf `**Verbindung testen**` um zu sehen, ob wir erfolgreich mit der Datenbank verbinden können. Wenn ja, klicke auf **`Ressourcen speichern`**, ansonsten überprüfe doppelt, ob der Hostname, der Port, der Benutzername und das Passwort korrekt sind.

Nach dem Erstellen einer Ressource wird die bereitgestellte REST-API wie angezeigt angezeigt.

![rest_ex_ready](https://cdn.illacloud.com/official-website/img/official-site/components/rest_ex_ready.jpg)

-   Im Builder erstellen

Melde dich bei deinem Illa Account an, erstelle ein Projekt in Illa Builder auf der `**App**` Seite und navigieren Sie zur **`Action List`** am unteren Ende der Seite. Klicke auf `**new**`, dann wähle **`REST API`** aus der Datenbankliste. Dann verbinden Sie sich mit der Datenbank mit den erforderlichen Parametern, die in `Verbindungseinstellungen` unten beschrieben sind.

Klicke auf `**Verbindung testen**` um zu sehen, ob wir erfolgreich mit der Datenbank verbinden können. Wenn ja, klicke auf **`Ressourcen speichern`**, ansonsten überprüfe doppelt, ob der Hostname, der Port, der Benutzername und das Passwort korrekt sind.

### Verbindungseinstellungen

Hier müssen wir Informationen zur Verbindung mit der REST API-Datenbank bereitstellen.

![rest_config](https://cdn.illacloud.com/official-website/img/official-site/components/rest_config.jpg)

| Eigenschaften     | Benötigt | Beschreibung                                                                                         |
| ----------------- | -------- | ---------------------------------------------------------------------------------------------------- |
| Name              | benötigt | Der Name für die Ressource beim Erstellen von Aktionen in der ILLA.                                  |
| Basis-URL         | benötigt | Die URL oder IP-Adresse für Ihre Datenbank                                                           |
| URL-Parameter     | optional | zusätzliche Informationen bereitstellen oder das Verhalten von API-Anfragen ändern                   |
| Kopfzeilen        | optional | zusätzliche Metadaten über die API-Anfrage oder Antwort                                              |
| Kekse             | optional | kleine Datenmengen, die auf der Client-Seite gespeichert und mit jeder API-Anfrage gesendet werden   |
| Authentifizierung | optional | Überprüfung der Identität des Benutzers oder der Anwendung, die die API-Anfrage stellt               |

### Aktionen erstellen

Wir haben eine REST-API-Ressource erstellt; wir können die Aktion hinzufügen, indem wir REST API aus der Aktionsliste auswählen und die **`Aktion erstellen`** Schaltfläche auswählen.

![rest_resource_list](https://cdn.illacloud.com/official-website/img/official-site/components/rest_resource_list.jpg)

Jetzt haben wir den REST API Server als eine Aktion zu unserer Build-Seite hinzugefügt.

![rest](https://cdn.illacloud.com/official-website/img/official-site/components/rest.jpg)

### REST-API konfigurieren

| Eigenschaften    | Beschreibung                                                                                                                           |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| Aktionstyp       | GET, POST, PUT, HEAD, PATCH, DELETE, OPTIONEN                                                                                          |
| URL-Parameter    | zusätzliche Informationen bereitstellen oder das Verhalten von API-Anfragen ändern, wenn nicht in der Verbindungseinstellung angegeben |
| Kopfzeilen       | zusätzliche Metadaten über die API-Anfrage oder Antwort, wenn nicht in der Verbindungseinstellung angegeben                            |
| Transformator    | Umwandlung von Daten in den Stil, den Sie mögen, mit JavaScript                                                                        |

#### ERHALTEN

Daten von einem Server abrufen oder abrufen. Bei einer GET-Anfrage gibt der Client die URL der Ressource an, auf die er zugreifen möchte, und der Server antwortet mit den angeforderten Daten.

#### POST

erstellen Sie eine neue Ressource auf dem Server. Es erlaubt es Clients, Daten auf den Server zu übermitteln und zu speichern. Beim Senden einer POST-Anfrage enthält der Client die Daten im Textteil der Anfrage.

#### PUT

aktualisiere oder ersetze eine vorhandene Ressource. Es wird in der Regel verwendet, um den Zustand oder den Inhalt einer Ressource zu ändern, die durch eine bestimmte URL identifiziert wird.

#### HEAD

ähnelt der GET-Methode, aber mit einem Schlüsselunterschied: Die HEAD-Methode holt nur die Header der Antwort, ohne den Antwortkörper.

#### PATCH

eine bestimmte Ressource aktualisieren. Es wird in der Regel verwendet, um partielle Änderungen an einer existierenden Ressource vorzunehmen, ohne die gesamte Ressource zu ersetzen.

#### LÖSCHEN

die Löschung einer bestimmten Ressource oder Daten durch eine angegebene URL anfordern

### Gehäuse verwenden

Wir zeigen Ihnen, wie Sie die Formularinformationen aus REST API-Daten automatisch ausfüllen.

#### Schritt 1 Eine Aktion hinzufügen

Lassen Sie uns eine Tabelle in REST API mit den Spalten `id`, `name` und `age` erstellen. Das Id 1 Dokument ist, was wir in der Formularkomponente zeigen wollen.

![rest_get_all](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_all.jpg)

Dann können wir eine neue Aktion für REST API aus der Aktionsliste in der App, die wir erstellen möchten, erstellen enthält alle Daten im Bucket und den Namen restapi1.

Wähle `GET` für `**Action Type**` aus. Geben Sie in URL-Parametern `id` für Schlüssel und `1` für Wert ein**.**

Klicke **`Speichern`** und **`Run`** um diese Aktion zu aktivieren.

![rest_get_1](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_1.jpg)

#### Schritt 2 Komponenten hinzufügen

Füge eine **`form`** Komponente mit dem Standardtext "Form" und einem "Absenden"-Button hinzu.

Füge eine **`input`** Komponente hinzu, eine **`number input`** Komponente und eine **`button`** Komponente von `**Einfügen**` in das Formular ein.

Bezeichnen Sie die Eingabekomponente für "Name", die Zahleneingabekomponente für "Alter" und die Schaltflächenkomponente für "Automatische Füllung".

Legen Sie den Platzhalter der Eingabekomponente auf "Bitte geben Sie Ihren Namen hier ein" und den Standardwert der Zahleneingabe auf 0 wie unten dargestellt.

![rest_layout](https://cdn.illacloud.com/official-website/img/official-site/components/rest_layout.jpg)

#### Schritt 3 die Komponente konfigurieren

Für die "Auto Fill" **`button`** Komponenten können wir sie so konfigurieren, dass sie den Wert in **`input`** Komponente festlegen.

1.  Klicken Sie auf die Komponente **`Schaltfläche`** um die Prüfungsseite zu öffnen. Unter **`Event handler`** klicke `**+ Neu**`.
2.  Wählen Sie im Editier-Ereignis-Handler **`Control component`** in Aktion und wählen Sie die **`input`** Komponente aus, die Sie als Ziel der Veranstaltung aktualisieren möchten.
3.  Wähle die **`setValue`** Aktion und setze den Wert als `{{restapi1.data[0]["name"]}}`

![rest_get_input](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_input.jpg)

In ähnlicher Weise können für "Auto Fill" **`button`** Komponenten Werte in der **`number input`** Komponente gesetzt werden.

1.  Unter **`Event handler`** klicke `**+ Neu**`.
2.  Wählen Sie im Editier-Ereignis-Handler **`Control component`** in Aktion und wählen Sie die **`number input`** Komponente aus, die Sie als Ziel der Veranstaltung aktualisieren möchten.
3.  Wähle die **`setValue`** Aktion und setze den Wert als `{{restapi1.data[0]["age"]}}`

![rest_get_num](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_num.jpg)

Für die "Absenden" **`button`** Komponente können wir es so konfigurieren, dass es das Formular abschickt, wenn das Formular ausgefüllt ist

1.  Klicke auf die **`Schaltfläche`** Komponente, um die Prüfungsseite zu öffnen. Unter **`Event handler`** klicke `**+ Neu**`.
2.  Wählen Sie im Editier-Ereignis-Handler **`Control component`** in Aktion und wählen Sie die **`form`** Komponente aus, die Sie als Ziel des Ereignisses aktualisieren möchten.
3.  Wähle die **`submit`** Methode

![rest_submit_config](https://cdn.illacloud.com/official-website/img/official-site/components/rest_submit_config.jpg)

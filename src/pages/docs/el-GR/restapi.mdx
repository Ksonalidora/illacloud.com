## <h2 hidden>Τι είναι REST API?</h2>

Στο ILLA Cloud, η λειτουργικότητα REST API εξουσιοδοτεί τους προγραμματιστές να αλληλεπιδρούν με εξωτερικά συστήματα, υπηρεσίες και πηγές δεδομένων με τη χρήση του αρχιτεκτονικού στυλ διαβίβασης του αντιπροσωπευτικού κράτους (REST). Με το API REST στο ILLA Cloud, οι προγραμματιστές μπορούν να ενσωματώσουν απρόσκοπτα τις εφαρμογές τους χαμηλού κώδικα με άλλες εφαρμογές, βάσεις δεδομένων ή υπηρεσίες.

Το REST API στο ILLA Cloud επιτρέπει τη δημιουργία προσαρμοσμένων τελικών σημείων που μπορούν να χειριστούν αιτήματα HTTP, όπως GET, POST, PUT και DELETE. Αυτό επιτρέπει στους προγραμματιστές να εκτελούν διάφορες λειτουργίες, συμπεριλαμβανομένης της ανάκτησης δεδομένων, της υποβολής δεδομένων, της ενημέρωσης εγγραφών και της διαγραφής πληροφοριών.

### Δημιουργία API REST

Υπάρχουν δύο τρόποι για να δημιουργήσετε έναν πόρο στην Illa μετά την είσοδό σας στο λογαριασμό Illa.

-   Δημιουργία σε πόρους

Συνδεθείτε στον Illa λογαριασμό σας, επιλέξτε `**Πόροι**` στην κορυφή της σελίδας και κάντε κλικ στο κουμπί `**Δημιουργία Νέου**`.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Επιλέξτε **`REST API`** από τη λίστα της βάσης δεδομένων.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Συνδεθείτε στη βάση δεδομένων με τις απαιτούμενες παραμέτρους που περιγράφονται στις παρακάτω ρυθμίσεις σύνδεσης.

Κάντε κλικ στο κουμπί `**Δοκιμή Σύνδεσης**` για να δείτε αν μπορούμε να συνδεθείτε με επιτυχία στη βάση δεδομένων. Εάν ναι, κάντε κλικ στο **`Αποθήκευση πόρων`**, αλλιώς, ελέγξτε ξανά το όνομα κεντρικού υπολογιστή, τη θύρα, το όνομα χρήστη και τον κωδικό πρόσβασης είναι σωστά.

Μετά τη δημιουργία ενός πόρου, το έτοιμο API REST θα εμφανίζεται όπως εμφανίζεται.

![rest_ex_ready](https://cdn.illacloud.com/official-website/img/official-site/components/rest_ex_ready.jpg)

-   Δημιουργία στο Builder

Συνδεθείτε στον Illa λογαριασμό σας, δημιουργήστε ένα έργο στο Illa Builder στη σελίδα `**App**`, και πλοηγηθείτε στο **`Λίστα Ενεργειών`** στο κάτω μέρος της σελίδας. Κάντε κλικ στο κουμπί `**new**`, και μετά επιλέξτε **`REST API`** από τη λίστα της βάσης δεδομένων. Στη συνέχεια, συνδεθείτε στη βάση δεδομένων με τις απαιτούμενες παραμέτρους που περιγράφονται στις "Ρυθμίσεις Σύνδεσης" παρακάτω.

Κάντε κλικ στο κουμπί `**Δοκιμή Σύνδεσης**` για να δείτε αν μπορούμε να συνδεθείτε με επιτυχία στη βάση δεδομένων. Εάν ναι, κάντε κλικ στο **`Αποθήκευση πόρων`**, αλλιώς, ελέγξτε ξανά το όνομα κεντρικού υπολογιστή, τη θύρα, το όνομα χρήστη και τον κωδικό πρόσβασης είναι σωστά.

### Ρύθμιση REST API

| Ιδιότητες         | Περιγραφή                                                                                                                     |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| Τύπος Ενέργειας   | GET, POST, PUT, HEAD, PATCH, DELETE, OPTIONS                                                                                  |
| Παράμετρος URL    | παρέχει πρόσθετες πληροφορίες ή να τροποποιήσει τη συμπεριφορά των αιτημάτων API εάν δεν καθορίζεται στη ρύθμιση σύνδεσης     |
| Κεφαλίδες         | πρόσθετα μεταδεδομένα σχετικά με την αίτηση ή την απάντηση API εάν δεν καθορίζονται στη ρύθμιση σύνδεσης                      |
| Μετασχηματιστής   | μετατρέποντας τα δεδομένα σε στυλ που σας αρέσει χρησιμοποιώντας JavaScript                                                   |

#### GET

ανάκτηση ή λήψη δεδομένων από ένα διακομιστή. Όταν κάνετε ένα αίτημα GET, ο πελάτης καθορίζει τη διεύθυνση URL του πόρου που θέλει να έχει πρόσβαση, και ο διακομιστής ανταποκρίνεται με τα ζητούμενα δεδομένα.

#### POST

δημιουργήστε έναν νέο πόρο στον διακομιστή. Επιτρέπει στους πελάτες να υποβάλλουν δεδομένα στο διακομιστή για επεξεργασία και αποθήκευση. Κατά την αποστολή ενός αιτήματος POST, ο πελάτης περιλαμβάνει τα δεδομένα στο σώμα του αιτήματος.

#### PUT

ενημέρωση ή αντικατάσταση υπάρχοντος πόρου. Συνήθως χρησιμοποιείται για την τροποποίηση της κατάστασης ή του περιεχομένου ενός πόρου που προσδιορίζεται από μια συγκεκριμένη διεύθυνση URL.

#### HEAD

παρόμοια με τη μέθοδο GET, αλλά με μια βασική διαφορά: η μέθοδος HEAD ανακτά μόνο τις κεφαλίδες της απάντησης, χωρίς το σώμα απόκρισης.

#### PATCH

ενημέρωση ενός συγκεκριμένου πόρου. Συνήθως χρησιμοποιείται για να κάνει μερικές τροποποιήσεις σε έναν υπάρχοντα πόρο χωρίς να αντικαταστήσει ολόκληρο τον πόρο.

#### ΔΙΑΓΡΑΦΗ

Αίτηση κατάργησης συγκεκριμένου πόρου ή δεδομένων που προσδιορίζονται από μια δοσμένη διεύθυνση URL

### Περίπτωση χρήσης

Θα αποδείξουμε πώς να συμπληρώσετε αυτόματα τις πληροφορίες φόρμας από τα δεδομένα REST API.

#### Βήμα 1 Προσθέστε μια ενέργεια

Επιτρέψτε μας να δημιουργήσουμε έναν πίνακα στο REST API με στήλες `id`, `name`, και `age`. Το έγγραφο id 1 είναι αυτό που θέλουμε να δείξουμε στο στοιχείο της φόρμας.

![rest_get_all](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_all.jpg)

Στη συνέχεια, μπορούμε να δημιουργήσουμε μια νέα ενέργεια για το REST API από τη λίστα ενεργειών στην εφαρμογή που θέλουμε να οικοδομήσουμε, απαριθμεί όλα τα δεδομένα στον κάδο και ονομάζεται restapi1.

Επιλέξτε `GET` για `**Είδος Ενέργειας**`. Στις Παράμετροι URL, βάλτε `id` για κλειδί και `1` για τιμή**.**

Πατήστε **`Save`** και **`Run`** για να ενεργοποιήσετε αυτήν την ενέργεια.

![rest_get_1](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_1.jpg)

#### Βήμα 2 Προσθήκη Συστατικών

Πρόσθεσε ένα στοιχείο **`form`** με το προεπιλεγμένο κείμενο "Form" και ένα κουμπί "Submit" σε καμβά.

Πρόσθεσε ένα **`input`** component, ένα **`number input`** component, και ένα **`button`** component από `**Insert**` στη φόρμα.

Επισημάνετε το στοιχείο εισόδου να είναι "Όνομα", το στοιχείο εισόδου αριθμού να είναι "Ηλικία" και το στοιχείο κουμπί να είναι "Αυτόματη πλήρωση".

Ορίστε το σύμβολο κράτησης θέσης του στοιχείου εισόδου σε "Παρακαλώ εισάγετε το όνομά σας εδώ" και την προεπιλεγμένη τιμή εισόδου του αριθμού στο 0, όπως φαίνεται παρακάτω.

![rest_layout](https://cdn.illacloud.com/official-website/img/official-site/components/rest_layout.jpg)

#### Βήμα 3 Διαμόρφωση του εξαρτήματος

Για την εφαρμογή "Αυτόματη συμπλήρωση" **`button`** components, μπορούμε να την ρυθμίσουμε ώστε να ορίζει την τιμή σε **`input`** component.

1.  Κάντε κλικ στο στοιχείο "Auto Fill" **`button`** για να ανοίξετε τη σελίδα επιθεώρησης της. Κάτω από **`Χειριστής Συμβάντων`**, κάντε κλικ στο `**+ Νέο**`.
2.  Στο χειριστή συμβάντων επεξεργασίας, επιλέξτε **`Control component`** εν ενεργεία, και επιλέξτε το στοιχείο **`input`** που θέλετε να ενημερώσετε ως στόχο του γεγονότος.
3.  Επιλέξτε την **`setValue`** ενέργεια και ορίστε την τιμή ως `{{restapi1.data[0]["name"]}}`

![rest_get_input](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_input.jpg)

Ομοίως, για τα συστατικά "Auto Fill" **`button`** για να ορίσετε τιμή σε **`number input`** component.

1.  Κάτω από **`Χειριστής Συμβάντων`**, κάντε κλικ στο `**+ Νέο**`.
2.  Στο χειριστή συμβάντων επεξεργασίας, επιλέξτε **`Control component`** εν ενεργεία, και επιλέξτε το **`number input`** component που θέλετε να ενημερώσετε ως στόχο του γεγονότος.
3.  Επιλέξτε την **`setValue`** ενέργεια και ορίστε την τιμή ως `{{restapi1.data[0]["age"]}}`

![rest_get_num](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_num.jpg)

Για το στοιχείο "Υποβολή" **`button`** της εφαρμογής, μπορούμε να το ρυθμίσουμε ώστε να υποβάλουμε τη φόρμα αν η φόρμα ολοκληρωθεί

1.  Κάντε κλικ στο στοιχείο **`button`** για να ανοίξετε τη σελίδα ελέγχου. Κάτω από **`Χειριστής Συμβάντων`**, κάντε κλικ στο `**+ Νέο**`.
2.  Στο χειριστή συμβάντων επεξεργασίας, επιλέξτε **`Control component`** εν ενεργεία, και επιλέξτε το στοιχείο **`form`** που θέλετε να ενημερώσετε ως στόχο του γεγονότος.
3.  Επιλέξτε τη μέθοδο **`submit`**

![rest_submit_config](https://cdn.illacloud.com/official-website/img/official-site/components/rest_submit_config.jpg)

### Configura REST API

| Propriet√†          | Descrizione                                                                                                                           |

| Tipo Di Azione     | GET, POST, PUT, HEAD, PATCH, DELETE, OPZIONI                                                                                          |

#### OTTA

recuperare o recuperare dati da un server. Quando si effettua una richiesta GET, il client specifica l'URL della risorsa che desidera accedere e il server risponde con i dati richiesti.

#### POST

crea una nuova risorsa sul server. Permette ai client di inviare dati al server per essere elaborati e memorizzati. Quando si invia una richiesta POST, il cliente include i dati nel corpo della richiesta.

#### PUT

aggiornare o sostituire una risorsa esistente. Viene tipicamente utilizzato per modificare lo stato o il contenuto di una risorsa identificata da un URL specifico.

#### TESTA

simile al metodo GET, ma con una differenza fondamentale: il metodo HEAD recupera solo le intestazioni della risposta, senza il corpo di risposta.

#### PERCORSO

aggiornare una risorsa specifica. √à tipicamente utilizzato per apportare modifiche parziali a una risorsa esistente senza sostituire l'intera risorsa.

#### ELIMINA

richiedere la rimozione di una risorsa o dati specifici identificati da un determinato URL

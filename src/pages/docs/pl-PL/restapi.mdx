## <h2 hidden>Co to jest REST API?</h2>

W ILLA Cloud funkcja REST API umożliwia programistom interakcję z zewnętrznymi systemami, usługi i źródła danych poprzez wykorzystanie stylu architektonicznego transferu państwa przedstawicielskiego (REST). Dzięki REST API w chmurze ILLA deweloperzy mogą bezproblemowo zintegrować swoje aplikacje o niskim kodzie z innymi aplikacjami, bazami danych lub usługami.

Interfejs API REST w chmurze ILLA pozwala na tworzenie własnych punktów końcowych, które mogą obsługiwać żądania HTTP, takie jak GET, POST, PUT, DELETE. Umożliwia to programistom wykonywanie różnych operacji, w tym pobieranie danych, przesyłanie danych, aktualizowanie rekordów i usuwanie informacji.

### Utwórz REST API

Istnieją dwa sposoby na utworzenie zasobu w Illa po zalogowaniu się na swoje konto Illa.

-   Utwórz w zasobach

Zaloguj się do swojego konta Illa wybierz `**Zasoby**` na górze strony i kliknij przycisk `**Utwórz nowy**`.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Wybierz **`REST API`** z listy baz danych.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Połącz się z bazą danych z wymaganymi parametrami opisanymi poniżej w "Ustawienia połączenia".

Kliknij `**Test Connection**` aby sprawdzić, czy możemy połączyć się z bazą danych. Jeśli tak, kliknij **`Zapisz zasoby`**, sprawdź dwukrotnie nazwę hosta, port, nazwę użytkownika i hasło jest poprawne.

Po utworzeniu zasobu, gotowe API REST będzie wyświetlane jak pokazano.

![rest_ex_ready](https://cdn.illacloud.com/official-website/img/official-site/components/rest_ex_ready.jpg)

-   Utwórz w Konstruktorze

Zaloguj się na swoje konto Illa, utwórz projekt w Illa Builder na stronie `**App**`, i przejdź do **listy działań\`** na dole strony. Kliknij `**nowe**`, a następnie wybierz **`REST API`** z listy bazy danych. Następnie połącz się z bazą danych z wymaganymi parametrami opisanymi w `Connection Settings` poniżej.

Kliknij `**Test Connection**` aby sprawdzić, czy możemy połączyć się z bazą danych. Jeśli tak, kliknij **`Zapisz zasoby`**, sprawdź dwukrotnie nazwę hosta, port, nazwę użytkownika i hasło jest poprawne.

### Ustawienia połączenia

Tutaj musimy podać informacje do połączenia z bazą danych REST API.

![rest_config](https://cdn.illacloud.com/official-website/img/official-site/components/rest_config.jpg)

| Właściwości                | Wymagane     | Opis                                                                                     |
| -------------------------- | ------------ | ---------------------------------------------------------------------------------------- |
| Nazwisko                   | wymagane     | Nazwa zasobu podczas tworzenia akcji w ILLA.                                             |
| Podstawowy adres URL       | wymagane     | Adres URL lub IP dla Twojej bazy danych                                                  |
| Parametry URL              | fakultatywne | dostarcza dodatkowych informacji lub modyfikuje zachowanie żądań API                     |
| Nagłówki                   | fakultatywne | dodatkowe metadane dotyczące żądania lub odpowiedzi API                                  |
| Pliki cookie               | fakultatywne | małe dane przechowywane po stronie klienta i wysyłane z każdym zapytaniem API            |
| Uwierzytelnianie           | fakultatywne | weryfikacja tożsamości użytkownika lub aplikacji składającej wniosek API                 |

### Konfiguruj REST API

| Właściwości     | Opis                                                                                                                   |
| --------------- | ---------------------------------------------------------------------------------------------------------------------- |
| Typ akcji       | GET, POST, PUT, HEAD, PATCH, DELETE, OPCJE                                                                             |
| Parametr URL    | podaj dodatkowe informacje lub modyfikuj zachowanie żądań API, jeśli nie określono w ustawieniach połączenia           |
| Nagłówki        | dodatkowe metadane dotyczące żądania lub odpowiedzi API, jeśli nie określono w ustawieniach połączenia                 |
| Transformator   | przekształcanie danych w styl jaki lubisz za pomocą JavaScript                                                         |

#### POBIERZ

pobierz lub pobierz dane z serwera. Przy składaniu żądania GET klient określa adres URL zasobu, do którego chce uzyskać dostęp, a serwer odpowiada wymaganym danym.

#### POST

utwórz nowy zasób na serwerze. Pozwala klientom na przesyłanie danych do serwera do przetwarzania i przechowywania. Podczas wysyłania żądania POST klient zawiera dane w treści żądania.

#### PT

aktualizowanie lub zastępowanie istniejącego zasobu. Jest on zazwyczaj używany do modyfikowania stanu lub zawartości zasobu zidentyfikowanego przez konkretny adres URL.

#### GŁÓWNY

Podobnie jak w przypadku metody GET, ale z jedną kluczową różnicą: metoda HEAD pobiera tylko nagłówki odpowiedzi, bez zawartości odpowiedzi.

#### PATRZ

zaktualizuj określony zasób. Jest on zazwyczaj wykorzystywany do dokonywania częściowych modyfikacji istniejącego zasobu bez zastępowania całego zasobu.

#### USUŃ

zażądaj usunięcia określonego zasobu lub danych zidentyfikowanych przez podany adres URL

### Przypadek użycia

Pokażemy jak automatycznie wypełniać dane formularza z danych REST API.

#### Krok 1 Dodaj akcję

Utwórzmy tabelę w REST API z kolumnami `id`, `name`, i `age`. Dokument id 1 jest tym, co chcemy pokazać w komponencie formularza.

![rest_get_all](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_all.jpg)

Następnie możemy utworzyć nową akcję dla REST API z listy akcji w aplikacji, którą chcesz zbudować, wykaz wszystkich danych w wiadrze i nazwany restapi1.

Wybierz `GET` dla `**Typ akcji**`. W parametrach URL wpisz `id` dla klucza i `1` dla wartości**.**

Kliknij **`Zapisz`** i **Uruchom\`**, aby aktywować tę akcję.

![rest_get_1](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_1.jpg)

#### Krok 2 Dodaj komponenty

Dodaj komponent **form\`** z domyślnym tekstem "Form" i przyciskiem "Wyślij" do płótna.

Dodaj komponent **`input`**, komponent **liczba\`** i komponent **`przycisk`** z `**Insert**` do formularza.

Etykietuj komponent wejściowy jako "Nazwa", komponent wejściowy liczb ma być "Wiek" i komponent przycisku, który ma być "Automatyczne wypełnianie".

Ustaw symbol zastępczy komponentu wejściowego na "Proszę wpisać swoją nazwę tutaj" i domyślną wartość wejścia numeru na 0 jak pokazano poniżej.

![rest_layout](https://cdn.illacloud.com/official-website/img/official-site/components/rest_layout.jpg)

#### Krok 3 Skonfiguruj komponent

Dla komponentów "Auto Fill" **`button`** możemy je skonfigurować tak, aby ustawić wartość w komponencie **`input`**.

1.  Kliknij komponent "Auto Fill" **`button`**, aby otworzyć jego stronę kontrolną. W **`Obsługa zdarzeń`**, kliknij `**+ Nowa**`.
2.  W obsłudze zdarzeń edytuj wybierz **`składnik kontroli`** w akcji, i wybierz komponent **input\`**, który chcesz zaktualizować jako cel wydarzenia.
3.  Wybierz działanie **`setValue`** i ustaw wartość jako `{{restapi1.data[0]["name"]}}`

![rest_get_input](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_input.jpg)

Podobnie, dla komponentów "Auto Fill" **`button`** aby ustawić wartość w komponencie **`number input`**.

1.  W **`Obsługa zdarzeń`**, kliknij `**+ Nowa**`.
2.  W obsłudze zdarzeń edytuj wybierz **`składnik kontroli`** w akcji, i wybierz komponent **numer wejścia\`**, który chcesz zaktualizować jako cel wydarzenia.
3.  Wybierz działanie **`setValue`** i ustaw wartość jako `{{restapi1.data[0]["age"]}}`

![rest_get_num](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_num.jpg)

Dla komponentu "Prześlij" **`button`** możemy go skonfigurować do przesłania formularza, jeśli formularz jest wypełniony

1.  Kliknij komponent **`przycisk`, aby otworzyć jego stronę kontrolną. W **`Obsługa zdarzeń`**, kliknij \`**+ Nowa\*\*\`.
2.  W obsłudze zdarzeń edytuj wybierz **`składnik kontroli`** w akcji, i wybierz komponent **formularz\`**, który chcesz zaktualizować jako cel wydarzenia.
3.  Wybierz metodę **`zgłoś`**

![rest_submit_config](https://cdn.illacloud.com/official-website/img/official-site/components/rest_submit_config.jpg)

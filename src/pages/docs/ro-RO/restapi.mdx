## <h2 hidden>Ce este REST API?</h2>

În ILLA Cloud, funcționalitatea REST API permite dezvoltatorilor să interacționeze cu sistemele externe. servicii și surse de date prin utilizarea stilului arhitectural al transferului statului de reprezentare (REST). Cu REST API în ILLA Cloud, dezvoltatorii pot integra perfect aplicațiile lor cu coduri joase cu alte aplicații, baze de date sau servicii.

REST API în ILLA Cloud permite crearea de criterii personalizate care pot trata cereri HTTP, cum ar fi GET, POST, PUT și DELETE. Acest lucru permite dezvoltatorilor să efectueze diverse operațiuni, inclusiv extragerea de date, trimiterea de date, actualizarea înregistrărilor și ștergerea informațiilor.

### Creează API REST

Există două modalități de a crea o resursă în Illa după ce te autentifici în contul tău Illa.

-   Creează în resurse

Conectează-te la contul tău Illa, selectează `**Resurse**` din partea de sus a paginii și apasă butonul `**Creează nou**`.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Selectaţi **`REST API`** din lista de baze de date.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Conectați-vă la baza de date cu parametrii necesari descriși în `Setări conexiuni` de mai jos.

Faceți clic pe `**Test Connection**` pentru a vedea dacă ne putem conecta cu succes la baza de date. Dacă da, apasă pe **`Salvează Resurse`**, în caz contrar, verifică de două ori numele gazdei, portul, numele de utilizator și parola sunt corecte.

După crearea unei resurse, API-ul REST gata va fi afișat ca afișat.

![rest_ex_ready](https://cdn.illacloud.com/official-website/img/official-site/components/rest_ex_ready.jpg)

-   Creează în Constructor

Conectează-te la contul tău Illa, creează un proiect în Illa Builder în pagina `**App**`, şi navighează la **`Lista de Acţiuni`** în partea de jos a paginii. Faceţi clic pe `**nou**`, apoi selectaţi **`REST API`** din lista de baze de date. Apoi, conectați-vă la baza de date cu parametrii necesari descriși în `Setări conexiuni` de mai jos.

Faceți clic pe `**Test Connection**` pentru a vedea dacă ne putem conecta cu succes la baza de date. Dacă da, apasă pe **`Salvează Resurse`**, în caz contrar, verifică de două ori numele gazdei, portul, numele de utilizator și parola sunt corecte.

### Configurare REST API

| Proprietăți      | Descriere                                                                                                                       |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| Tip de acțiune   | GET, POST, PUT, HEAD, PATCH, DELETE, OPȚIUNI                                                                                    |
| Parametru URL    | furnizați informații suplimentare sau modificați comportamentul solicitărilor API dacă nu este specificat în setarea conexiunii |
| Antete           | metadate suplimentare despre cererea sau răspunsul API dacă nu este specificat în setarea conexiunii                            |
| Transformator    | Transformarea datelor în stilul care îți place să folosești JavaScript                                                          |

#### OBȚINE

preia sau preia datele de pe un server. Atunci când face o cerere GET, clientul specifică URL-ul resursei pe care dorește să o acceseze, iar serverul răspunde cu datele solicitate.

#### POS

creează o resursă nouă pe server. Permite clienților să trimită date la server pentru a fi procesate și stocate. Când se trimite o solicitare POST, clientul include datele în corpul cererii.

#### Pune

actualizarea sau înlocuirea unei resurse existente. Este folosit de obicei pentru a modifica starea sau conținutul unei resurse identificate printr-un URL specific.

#### BRICA

similar cu metoda GET, dar cu o diferență esențială: metoda HEAD recuperează doar antetul răspunsului, fără corpul de răspuns.

#### PLICURI

actualizează o resursă specifică. Este folosit de obicei pentru a face modificări parțiale la o resursă existentă, fără a înlocui întreaga resursă.

#### Şterge

solicită eliminarea unei resurse sau a unor date specifice identificate printr-un URL dat

### Utilizare majusculă

Vom demonstra cum să completăm automat informațiile formularului din datele REST API.

#### Pasul 1 Adăugare acţiune

Haideţi să creăm un tabel în REST API cu coloanele `id`, `name`, şi `age`. Documentul id 1 este ceea ce dorim să arătăm în forma componentei.

![rest_get_all](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_all.jpg)

Apoi putem crea o nouă acțiune pentru REST API din lista de acțiuni din aplicația pe care dorim să o construim, listarea tuturor datelor în găleată și denumită restapi1.

Selectați `GET` pentru `**Tipul de acțiune**`. În parametrii URL, puneți `id` pentru cheie și `1` pentru valoare**.**

Faceţi clic pe **`Salvează`** şi **`Run`** pentru a activa această acţiune.

![rest_get_1](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_1.jpg)

#### Pasul 2 Adaugă Componente

Adăugați o componentă **`form`** cu textul implicit "Form" și un buton "Trimite" pe planșă.

Adăugați o componentă **`input`**, o componentă **`number input`**, și o componentă **`button`** din `**Insert**` în formular.

Etichetați componenta de intrare pentru a fi "Nume", componenta de intrare a numărului pentru a fi "Age" și componenta butonului de a fi "Auto Fill".

Setați substituentul componentei de intrare la "Vă rugăm să puneți numele dvs. aici" și valoarea implicită a numărului introdus la 0 după cum se arată mai jos.

![rest_layout](https://cdn.illacloud.com/official-website/img/official-site/components/rest_layout.jpg)

#### Pasul 3 Configurarea componentei

Pentru componentele "Auto Fill" **`button`**, îl putem configura pentru a seta valoarea în **`input`** componentă.

1.  Faceți clic pe componenta "Auto Fill" **`button`** pentru a deschide pagina sa de inspecție. Sub **`Manipulatorul de evenimente`**, apasă `**+ New**`.
2.  În gestionarul de editare a evenimentului, selectați **`Componenta Control`** în acțiune, şi selectaţi componenta **`intrare`** pe care doriţi să o actualizaţi ca ţintă a evenimentului.
3.  Alegeți acțiunea **`setValue`** și setați valoarea ca `{{restapi1.data[0]["name"]}}`

![rest_get_input](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_input.jpg)

Similar, pentru componentele "Auto Fill" **`button`** pentru a seta valoarea în componenta **`number input`**.

1.  Sub **`Manipulatorul de evenimente`**, apasă `**+ New**`.
2.  În gestionarul de editare a evenimentului, selectați **`Componenta Control`** în acțiune, şi selectaţi componenta **`număr de introducere`** pe care doriţi să o actualizaţi ca ţintă a evenimentului.
3.  Alegeți acțiunea **`setValue`** și setați valoarea ca `{{restapi1.data[0]["age"]}}`

![rest_get_num](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_num.jpg)

Pentru componenta "Trimite" **`button`** o putem configura pentru a remite formularul dacă formularul este completat

1.  Faceți clic pe componenta **`button`** pentru a deschide pagina sa de inspecție. Sub **`Manipulatorul de evenimente`**, apasă `**+ New**`.
2.  În gestionarul de editare a evenimentului, selectați **`Componenta Control`** în acțiune, şi selectaţi componenta **`form`** pe care doriţi să o actualizaţi ca ţintă a evenimentului.
3.  Alege metoda **`submit`**

# <Heading hidden>ILLA的私有化部署</Heading>

私有化部署意味着在您自己的服务器或机器上部署和管理ILLA平台，而不是使用第三方服务提供商。这种方法通常被认为更安全和私密，因为您可以完全控制您的数据及其使用。

ILLA将提供云托管版本，并提供在各种平台上设置实例的选项。

## 私有化部署

在您自己的基础设施上私有化部署ILLA允许您将其与现有设置无缝集成，而无需维护单独的配置。这使您能够使用现有基础架构的特性和功能并将它们扩展到ILLA。在您的基础设施上部署ILLA的一些优势包括：

### 安全

通过在您自己的基础设施上部署ILLA，您可以将数据保存在本地，并确保您的API密钥或安全凭证永远不会离开您的服务器。这允许您对ILLA应用与您服务器上任何其他应用程序相同的安全规则。您使用ILLA构建的应用程序将托管在您的基础设施和公司网络内。您还可以选择在防火墙之后的内部网络上托管ILLA，并将其配置使用您自定义的域和安全策略。

### 内部数据库

通过私有化部署ILLA，您可以连接到在同一台机器或内网上运行的数据库，并使用它来构建应用程序。ILLA使用MongoDB存储应用程序数据，使用Redis存储会话信息和非关键缓存。如果您想将ILLA与部署在您基础设施上的自定义MongoDB或Redis服务器集成，您将需要修改环境变量的配置并将它们指向您自定义的MongoDB/Redis实例。

### 更新

通过私有化部署ILLA，您可以配置更灵活的更新计划和其他维护活动。您可以按照提供的步骤设置维护窗口并安排自动更新。这允许您以方便您和您的组织的方式维护和更新您的ILLA实例。

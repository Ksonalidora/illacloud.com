**在使用 ILLA CLI 进行本地化部署前，需先安装 docker. 在使用 ILLA CLI 时请将用户加入docker用户组或以root权限运行 ILLA CLI**

## 下载ILLA CLI
如果已经安装了Rust，你可以轻松通过**cargo**下载并安装ILLA CLI: 
``` 
> cargo install illa
```
**cargo**将会下载ILLA CLI及其依赖，并将ILLA CLI编译安装到cargo执行文件目录下，以便我们使用它.

安装完成后，我们可以通过**illa**命令使用ILLA CLI.

当然，你也可以通过以下链接下载对应操作系统的ILLA CLI可执行文件:

* [Windows](https://github.com/illa-family/illa/releases/latest/download/illa-x86_64-win.zip)
* [Linux](https://github.com/illa-family/illa/releases/latest/download/illa-x86_64-linux.tar.gz)
* [Mac](https://github.com/illa-family/illa/releases/latest/download/illa-x86_64-macos.tar.gz)

通过上述链接下载的ILLA CLI不同于通过**cargo**安装的ILLA CLI. 安装CLI后，你需要让它可执行. 以illa-linux为例:

```
# 下载illa命令行工具可执行文件
> wget https://github.com/illa-family/illa/releases/latest/download/illa-x86_64-linux.tar.gz

# 解压缩
> tar -zxvf illa-x86_64-linux.tar.gz

> cd illa-x86_64-linux

> chmod +x illa

# 运行illa命令行工具
> ./illa

```

## 检查本地化部署的前置依赖

**端口5432, 9999和8000需要为ILLA Builder保留，以免出现安装错误**

此命令将检查不同操作系统下ILLA Builder的前置依赖是否符合要求:
```
> illa doctor
```

## 部署ILLA Builder
ILLA CLI安装完成后，你可以通过其进行本地化部署和艾拉云部署.

**艾拉云期待为您服务**

只需要运行此命令即可在本地部署ILLA Builder: 

```
# ILLA Builder的访问端口可以更改
# 可将SERVER_ADDRESS设置为云服务器的公网ip(其默认值为localhost)以实现云服务器部署
> illa deploy --self --port=10000 --server-addr=<SERVER_ADDRESS>
```
现在你可以通过 http://localhost:10000 访问ILLA Builder.

## 更新 ILLA Builder
使用此命令可将ILLA Builder更新至最新发布版本:
```
> illa update --self
```

## 其他操作
```
# 列出ILLA Builder相关信息
> illa list --self

# 停止指定ILLA Builder
> illa stop --id=<ILLA_BUILDER_CONTAINER_ID> --self

# 移除指定ILLA Builder并删除对应持久化数据
> illa remove --id=<ILLA_BUILDER_CONTAINER_ID> --self

# 重启指定ILLA Builder
> illa restart --id=<ILLA_BUILDER_CONTAINER_ID> --self

# 帮助信息
> illa help
```
